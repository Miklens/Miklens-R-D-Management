# Final Architecture: Trials, Observations, Theme, and Deployment

## 1. Missing Modules: Field Trials, Lab Tests, Observations

### Overview
While "Experiments" cover the core scientific endeavor, the SRS specifies dedicated modules for `Field Trials`, `Laboratory Tests`, and `Observations`.
- **Field Trials**: Specifically tracks large-scale testing outside the lab (e.g., crop fields, greenhouses). Includes location and climate data.
- **Laboratory Tests**: Focused strictly on controlled environment testing (e.g., petri dishes, spectrometry).
- **Observations**: A feed of qualitative notes taken by scientists during either trials or lab tests.

### Components
- `src/pages/FieldTrials.tsx`: Map-pin icons, location data, timeline tracking.
- `src/pages/LaboratoryTests.tsx`: Flask icons, specific lab equipment tied to tests.
- `src/pages/Observations.tsx`: A feed-style UI (like a timeline or comment thread) for quick scientific notes.

---

## 2. Global Contexts

### ThemeContext
The prompt requires `Dark Mode` and `Light Mode`.
- **`src/contexts/ThemeContext.tsx`**: A React context that checks `localStorage` or `window.matchMedia('(prefers-color-scheme: dark)')` to set the initial theme. It toggles the `.dark` class on the root HTML element.

### Demo Auth Bypass
Currently, `AuthContext` expects valid Firebase credentials. To ensure the final delivered app is functional for reviewers right out of the box, we will add a fallback "Demo Login" feature. If the app detects placeholder API keys (`mock-api-key`), the login screen will offer a "Login as Demo User" button that sets a mock user state in context, bypassing the actual network request.

---

## 3. Deployment Configuration

### Firebase Hosting
- **`firebase.json`**: Defines the deployment rules. It will instruct Firebase CLI to deploy the `dist` folder generated by Vite as a Single Page Application (SPA), rewriting all requests to `/index.html` for React Router to handle.

### PWA and Clean Code
The application is now fully PWA-enabled with offline persistence. No further architecture changes are needed, but we must ensure all new pages are correctly integrated into the Sidebar and Router.
