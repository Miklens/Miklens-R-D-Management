# Phase 8, 10 & 11 Architecture: Experiments, Reports, and Analytics

## Phase 8: Experiments Module Architecture

### Overview
This module tracks scientific experiments tied to products. A core requirement is that failures increase knowledge rather than penalize users.

### Components
- **`src/pages/Experiments.tsx`**: A table view listing experiments. It will show the Experiment Number, Objective, Status (Success/Failure/Partial), Lead Scientist, and linked Product.
- **Visuals**: Uses Badges for status. A failure should not look "bad" (e.g., solid red) but rather informative (e.g., warning/amber or neutral gray to signify "Learning").

### Edge Cases
- **Data Density**: Experiments contain a lot of data (hypothesis, materials, readings). The list view must only show the summary, relying on a future detail view/modal for the rest.

## Phase 10: Reports Module Architecture

### Overview
Automated reporting. Employees shouldn't write long reports; AI generates them.

### Components
- **`src/pages/Reports.tsx`**: A list of generated reports (Weekly, Monthly, Product-specific) with download/export actions (PDF, Excel).
- **Design**: Card-based or list layout emphasizing the type of report and generation date.

### Edge Cases
- **Generation State**: Reports might take time to generate via AI. The UI must support a "Generating..." status.

---

## Phase 11: Analytics Module Architecture

### Overview
Deep dive charts for Management to understand performance, bottlenecks, and success rates.

### Components
- **`src/pages/Analytics.tsx`**: A grid of complex Recharts components. We need:
  - **BarChart**: Comparing Department Performance.
  - **PieChart**: Experiment Success/Failure Rates.
  - **LineChart**: Overall company progress or Innovation Index over time.

### Edge Cases
- **Responsiveness**: Charts must be wrapped in `ResponsiveContainer` to adapt to mobile or resized windows.
- **Dark Mode**: Recharts components need styling adjustments (axis colors, grid lines, tooltip backgrounds) to look good in both light and dark modes.
